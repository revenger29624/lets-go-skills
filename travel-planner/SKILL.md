# Travel Planner - 旅行攻略生成

## 核心功能

1. **多格式输入** - 支持文字描述、TXT、HTML、Word 文档作为旅行需求输入
2. **联网信息获取** - 查询目的地景点、美食、天气、交通
3. **结构化输出** - 生成格式规范的 Markdown 旅行攻略文档

## 输入格式处理

### 1. 纯文字输入
用户直接描述旅行需求，例如：
> "帮我规划一个去成都5天4晚的旅行，主要想看大熊猫和吃美食"

提取关键信息：目的地、天数、人数（若有）、预算（若有）、特殊兴趣偏好。

### 2. 文件输入（.txt / .html / .docx）

读取文件内容作为旅行需求上下文。使用对应工具读取：
- `.txt` → 直接读取文本
- `.html` → 提取可读文本（去除 HTML 标签）
- `.docx` → 读取 Word 文档内容

如果文件读取失败，提示用户提供纯文字描述或直接询问关键信息。

### 3. 联网搜索景点
使用 `online-search` 或 `web_fetch` 查询：
- 目的地热门景点（TOP10）
- 景点开放时间、门票、建议游玩时长
- 必吃美食、网红餐厅
- 交通方式（机场/火车站到市区）

### 4. 查询天气
使用 `weather` skill 查询目的地未来 3-7 天天气，给出穿衣建议。

## 工作流程

1. **收集需求** - 读取用户输入（文字或文件），提取目的地、天数、人数、偏好、预算
2. **联网搜索** - 查询景点、美食、天气、交通信息
3. **结构化整理** - 按输出模板组织内容
4. **生成 Markdown** - 输出完整攻略文件
5. **保存文件** - 使用 `write` 工具保存到工作目录，文件名格式：`[目的地]_旅行攻略_YYYYMMDD.md`

## 进阶技巧

- 用户给出模糊需求时，主动询问关键信息（天数、重点想玩什么）
- 如有多个目的地，按地理就近原则安排顺序
- 亲子游优先安排儿童友好景点，避免行程过满
- 情侣游可加入浪漫餐厅、夜景推荐
- 独自旅行增加安全提示和性价比推荐
